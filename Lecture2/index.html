Data types

A first look

x <- c(1, 3, 5)
y <- 5
months <- c("January", "February")

To create an object, that will remain in your workplace, you should:

  • 1. Give a (meaninful) name (e.g., x, y, months)
  • 2. Use an assignment operator (e.g., <-, =)
  • 3. Define your object (e.g., 5, c("January", "February"))

Matrix

Dimension names

  • You can set the row/column names either by using the 'dimnames' argument within the 'matrix' function or by using the 'colnames', 'rownames' functions.
x <- matrix(c(134, 134, 211, 5523), nrow = 2, ncol = 2)
rownames(x) <- c("ID1", "ID2")
colnames(x) <- c("Question1", "Question2")
x

##     Question1 Question2
## ID1       134       211
## ID2       134      5523

Matrix (Some drawbacks)

  • Combine data of a single type
  • Indexing not as easy, or intuitive, as in data frames (see below)
  • A series of useful functions, work better with data frames (see dplyr, ggplot2, packages etc.)
matrix(c(1, 2, "Man", "Woman"), 2, 2)

##      [,1] [,2]   
## [1,] "1"  "Man"  
## [2,] "2"  "Woman"

Lists

  • OK, so vectors have 1 dimensions, matrices 2 dimensions.
  • How many dimensions does a list have?

Data frames

  • The most useful, and relevant, data type.
  • Although it looks like a matrix, it can hold multiple data types
  • Popular packages (e.g., plyr, ggplot2) play well with data.frames
  • Sometimes it is just easier to convert a matrix to a data frame and move from there on (see below)
 
myDF = data.frame(ID = 1:10, condition = c(1, 2, 1, 2, 2, 1, 1, 2, 1, 1), 
Sex = c("M", "F", "M", "F", "M", "F", "M", "F", "M", "F"))
head(myDF)

##   ID condition Sex
## 1  1         1   M
## 2  2         2   F
## 3  3         1   M
## 4  4         2   F
## 5  5         2   M
## 6  6         1   F

myDF$liz = list("1", "2")
tail(myDF, n = 3)

##    ID condition Sex liz
## 8   8         2   F   2
## 9   9         1   M   1
## 10 10         1   F   2

Working directory

  • The folder that R operates
  • If you want to refer to data outside that directory, you need to give the relative, or full path of the file.

  • setwd()

Working directory

Things to look after

  • Seperate the differt folders in the path with either "/" or "\\". Do not use "\".
